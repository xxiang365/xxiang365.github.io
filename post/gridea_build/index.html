Created: Sep 17, 2020 12:33 AM

前言

有时候博客不需要一些花里胡哨的东西，最纯粹的博客可以简化为一个创作平台+一个评论系统（可选）。

Gridea是一种纯静态网站生成工具，非常适合不想麻烦折腾后端服务器，和后期维护的用户，类似的工具有Hugo，Hexo等，可以让注意力更加集中在内容创作上，非常适合我的需求。

二、具体操作

1, 以Github为博客的“后端”

  • 注册自己的github账号，注册成功后，新建仓库，用来存储网站的代码。
  • 申请 Personal access token，这样 Gridea 才能有权限将网站文件推送到你的仓库中。

    再依次选择 「Developer settings」→ 「Personal access tokens」→ 「Generate new token」
  • token 的权限我们只选择 Repo，申请成功，会给你一串数字+字母的信息，这个就是 Token。

2，以Gridea为本地前端，简单自由又安全

  • 安装好Gridea后，配置 Gridea，来连接 Github
  • 填写上面的信息后，点击 「检测远程连接」 按钮，如果提示 「远程连接成功」 那就ok啦

3，换成自己喜欢的域名（Optional）

  • 由于不是本篇的重点，就简单介绍一下，可以在freenom注册免费域名后，将注册域名的Nameservers设置为Cloudflare的dns解析服务器，需要等待一段时间生效。
  • 在Cloudflare中可以找到注册的域名，在DNS选项中添加一个CNAME解析，Content填写YourUserName.github.io即可。
  • 想要HTTPS访问，需要在Cloudflare和Github中设置SSL

3，Valine评论系统

  • 选择官网中的Pure主题（主题自带Valine评论系统前端的，并且可以设置暗色主题）

imhanjie/gridea-theme-pure

  • Valine评论系统前端包含在Gridea的主题-自定义配置中，只需要填入APP_ID和APP_KEY即可，APP_ID和APP_KEY是通过Valine云提供的评论后端

  • Valine云提供的评论后端

  • 手动在leancloud应用-存储中添加Comment，网友的评论可以在这里管理
